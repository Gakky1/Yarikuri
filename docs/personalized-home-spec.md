# 目的
初回の質問に答えたあと、ユーザーを「守るタイプ」「増やすタイプ」に分類しない。

ホーム画面には、
- 「守る」タブ
- 「増やす」タブ
の2つを必ず表示する。

それぞれのタブの中に出すカード内容は、初回質問の回答に応じて変える。

また、ホーム画面の上部には、その人に合った「今日のおすすめ」を1つ表示する。

既存UIはできるだけ活かしつつ、今より少し愛着が湧くように改善する。

# 初回質問
1. いちばん近い悩み
- 毎月あまりお金が残らない
- 何にお金を使っているか分からない
- 将来のお金がなんとなく不安
- 投資を始めたいけど怖い
- 貯金はあるが、増やし方が分からない

2. 毎月のお金の余裕
- ほぼ残らない
- 少しだけ残る
- ある程度残る
- かなり余裕がある
- よく分からない

3. 今ある支払い
- 特にない
- 住宅ローンがある
- 奨学金がある
- カードローンやリボ払いがある
- その他の借入がある
- 答えたくない

4. 急な出費に使えるお金
- ほぼない
- 1か月分未満
- 1〜3か月分くらい
- 3〜6か月分くらい
- 6か月分以上
- よく分からない

5. 今の生活に近いもの
- 一人暮らし
- パートナーと暮らしている
- 子どもがいる
- 親や家族を支えている
- 家族と同居している
- 答えたくない

6. 投資経験
- まったくない
- 興味はあるけど、まだ始めていない
- 積立だけしている
- 個別株や投資信託を少ししている
- ある程度慣れている

# やりたいこと
- 初回6問の回答を保存する
- 回答から userFinancialProfile を作る
- ホーム画面に「守る」と「増やす」の2つのタブを表示する
- 2つのタブは常に表示する
- ただし、各タブの中に出すカード内容は回答に応じて変える
- ホーム上部の todayRecommendation も回答に応じて変える

# 内部で持ちたい情報
- protectPriority
- spendingControlNeed
- debtCareNeed
- emergencyFundNeed
- growReadiness
- investmentConfidence

# 表示ルール
- ホーム画面には「守る」と「増やす」の2つのタブを表示する
- タブの順番は固定でよい
- 上下の優先順位で並べ替える考えは不要
- 「守る」タブの中身だけ、質問の回答に応じて変える
- 「増やす」タブの中身だけ、質問の回答に応じて変える
- todayRecommendation は、その人に今いちばん合う内容を1つ表示する

# 守るタブの出し分け
- spendingControlNeed が高いなら、支出見直し・固定費・使い道の見える化カードを出す
- debtCareNeed が高いなら、借入確認・返済優先度・リボ注意カードを出す
- emergencyFundNeed が高いなら、生活防衛資金・急な出費への備えカードを出す
- 子どもがいる、または家族を支えている場合は、家族向けの備えカードも候補に入れる

# 増やすタブの出し分け
- investmentConfidence が低いなら、投資の怖さを減らす・はじめて向け・少額スタートカードを出す
- investmentConfidence が中くらいなら、積立・NISA・続け方カードを出す
- investmentConfidence が高いなら、見直し・配分・投資整理カードを出す
- growReadiness が低い場合でも、増やすタブ自体は表示する
- ただし、無理に投資をすすめるより「準備から始める」内容を出す

# todayRecommendation の考え方
- その人に今いちばん合う行動を1つだけ表示する
- 難しい内容を並べず、すぐ動ける内容にする
- 例:
  - 固定費を1つ見直す
  - 使っていないサブスクを確認する
  - 急な出費に備える金額を確認する
  - 1000円でできる積立を考える
  - NISAの最初の1歩を見る

# UI改善の方向
- 既存UIをなるべく活かす
- ただし、少し愛着が湧く見た目にしたい
- やってほしいこと:
  - 文言を少しやわらかくする
  - タイトルや説明文を少し親しみやすくする
  - カードを少し見やすくする
  - 余白、角丸、アイコン、色味などを既存デザインに合う範囲でやさしく整える
  - 「怖い金融アプリ」ではなく「毎日少し見たくなる雰囲気」に近づける

# 制約
- 既存UIをできるだけ壊さない
- 既存の命名規則をなるべく守る
- 新しいライブラリ追加は最小限
- まずは大改造ではなく、最小変更で実装する
